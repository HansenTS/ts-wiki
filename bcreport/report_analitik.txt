# STANDARISASI SSRS REPORT **ANALITIK** UNTUK D365BC

## Report Analitik (Revenue per Product, Detail Stock Movement)
**Digunakan untuk analisis manajemen, sehingga harus rapi, profesional, dan mudah dibaca.**
- **Font Type:** Segoe UI
- **Header Section**
   - **Font Size:** 16 pt
   - **Bold:** Ya (untuk judul utama laporan, misalnya Revenue per Product Report)
   - **Italic:** Tidak digunakan (kecuali untuk catatan atau disclaimer kecil)
- **Column Header (tabel analitik)**
   - **Font Size:** 12 pt
   - **Bold:** Ya (agar membedakan dari detail data)
   - **Italic:** Tidak
- **Detail Data (isi tabel)**
   - **Font Size:** 10 pt
   - **Bold:** Hanya untuk subtotal/total baris
   - **Italic:** Tidak digunakan
- **Highlight/Emphasis**
  - Gunakan warna background ringan (light gray) untuk baris total agar lebih jelas
  - Bold untuk angka penting (misalnya total revenue, stock balance)


## Elemen Visual & Branding
**Logo Perusahaan**
- Selalu ambil dari Themes (centralized asset) agar konsisten di semua report.
- **Posisi:** kiri atas header report/dokumen.
- **Ukuran:** proporsional, jangan lebih dari 10–15% lebar halaman.
- **Warna & Style:**
   - Gunakan warna dari corporate theme (misalnya biru/abu-abu untuk otomotif).
   - Hindari warna mencolok untuk background tabel; gunakan light gray untuk highlight.

## Parameter input  
**(Date From – Date To)**
  - Posisi Sejajar : parameter tanggal harus berdampingan (misal : From di kiri, To di kanan)
  - Label konsisten : gunakan format label yang sama, "Tanggal Mulai" - "Tanggal Selesai" atau "From Date" - "To Date"
  - Format Tanggal seragam : dd/MM/YYYY (untuk indo), MM/dd/YYYY (untuk luar indo)


## Format Data
**Tanggal (Date)**
  - Format: DD-MMM-YYYY (contoh: 04-Feb-2026).
  - Konsisten di semua report, baik analitik maupun transaksi.
  - **Untuk jam (jika diperlukan):** DD-MMM-YYYY HH:mm.

**Angka (Number)**
  - Format umum: #,### (contoh: 12,345).
  - Untuk nilai uang: #,###.00 + simbol mata uang (Rp).
  - Persentase: ##.##%.
  - Hindari angka tanpa pemisah ribuan.

## Alignment
**Header Title:** Center alignment, Bold.
**Column Header:** Center alignment (agar tabel rapi).
**Detail Data:**
  - Text: Left alignment.
  - Number/Amount: Right alignment.
  - Date: Center alignment.
  - Footer/Disclaimer: Left alignment, italic jika catatan hukum.

## Font & Ukuran (Standar Umum)
**Font Type:**
  - Report Analitik → Segoe UI / Calibri (modern, clean).
  - Dokumen Transaksi → Arial / Times New Roman (formal, resmi).
**Ukuran Font:**
  - Judul Report/Dokumen: 14–16 pt, Bold.
  - Sub-header (info perusahaan, nomor dokumen): 11–12 pt.
**Column Header:** 11 pt, Bold.
**Detail Data:** 10 pt.
**Footer/Disclaimer:** 9–10 pt, italic jika catatan.

## Konsistensi Layout
**Margin:** 1 inch (2.54 cm) di semua sisi.
**Page Orientation:**
 - Analitik → Landscape (jika tabel lebar).
 - Transaksi → Portrait (lebih formal).
**Pagination:**
 - Tambahkan nomor halaman di footer: “Page X of Y”.
**Sectioning:**
 - Header → Logo + Judul.
 - Body → Data utama.
 - Footer → Catatan, tanda tangan, atau disclaimer.

## Warna & Efisiensi Cetak
  - **Minim warna blok :** hindari background warna 
  - **Highlight dengan garis tipis :** gunakan border tipis atau bold text untuk pemisahan, bukan di warna mencolok
