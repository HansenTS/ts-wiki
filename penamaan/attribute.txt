# PENAMAAN ATTRIBUTE DI D365CE
## PENAMAAN ATTRIBUTE
**Attribute Naming:** 
- **Format:** prefix_logicalname
- **Prefix:** gunakan publisher prefix organisasi (contoh: xti_,bsi_)
- **Panjang Maksimal:** 50 karakter(termasuk prefix)

**Contoh:** 
  - xti_creditlimit
  - xti_isgender (same day revision)
  - xti_contractstartdate (next release)

## Best Practice
1. Naming untuk Boolean: Gunakan prefix is, has, atau can (contoh: xti_isapproved, xti_haspermission)
2. Naming untuk Date: Akhiri dengan date atau datetime (contoh: xti_expirydate)
3. Naming untuk Lookup: Gunakan nama entitas yang direferensikan (contoh: xti_product)
4. Naming untuk Currency: Hindari kata "amount" jika sudah jelas dari konteks (contoh: xti_deposit lebih baik dari xti_depositamount)
5. Konsistensi Penamaan: Jika menggunakan istilah tertentu, gunakan secara konsisten di seluruh solution


## Checklist Validasi Penamaan Attribute
[  ] ✅Apakah menggunakan publisher prefix yang benar?
[  ] ✅Apakah nama sudah lowercase dengan underscore?
[  ] ✅Apakah nama cukup deskriptif dan mudah dipahami?
[  ] ✅Apakah mengikuti konvensi untuk tipe field tertentu (Boolean, Date, dll)?
[  ] ✅Apakah tidak menggunakan reserved words atau nama standard fields?
[  ] ✅Apakah konsisten dengan naming convention attribute lain dalam solution?
