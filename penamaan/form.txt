# PENAMAAN FORM DI D365CE
## PENAMAAN FORM
**Form Naming:**
  - **Format:** EntityName-FormType

**Form Types:**  
  - **Main Form:** EntityName – Main atau EntityName – Information
  - **Quick Create:** EntityName - Quick Create
  - **Quick View:** EntityName - Quick View
  - **Card Form:** EntityName – Card
  - **Custom/Specific Purpose:** EntityName - [Purpose]


**Contoh:**   
- Workorder – Main
- Workorder - Quick Create
- Workorder - Service Advisor View
- Service Request - Customer Portal
- Customer Feedback – Mobile


**Description:** Jelaskan tujuan publisher secara singkat  
        - Contoh: Publisher untuk solusi Technosoft Project Indonesia  

## Best Practice
1. Gunakan display name yang user-friendly (tidak perlu menyertakan prefix)
2. Sertakan purpose atau target audience jika ada multiple forms
3. Urutkan forms berdasarkan prioritas penggunaan
4. Konsisten dengan bahasa (Indonesia atau English) di seluruh workspace


## Checklist Validasi Penamaan Publisher
[  ] ✅ Format: EntityName - FormType atau EntityName - Purpose
[  ] ✅ Tidak menyertakan prefix di form name
[  ] ✅ Form type atau purpose jelas dan deskriptif
[  ] ✅ Konsisten dengan bahasa yang dipilih
[  ] ✅ Main form diberi nama yang jelas (Main atau Information)
[  ] ✅ Quick Create dan Quick View forms mengikuti naming convention
[  ] ✅ Custom forms menyertakan purpose atau target audience
